Information

To use this calculator:

  1. Have a spotter determine the distance and azimuth to the target location.
  2. Without moving, have the spotter determine the distance and azimuth to the artillery location.
  3. Input the two sets of values, and the computed result is the distance and azimuth from the artillery to the target.
  4. The optional left/right and drop/add correction can be entered as viewed from the spotter in meters. Negative values indicate correction to the left/closer to the spotter.
  5. Adjust the artillery settings to match the resulting values, then fire!

Target Reference Points (TRPs)

TRPs allow you to extend your spotting range beyond what binoculars can reach by chaining intermediate observation points together.

  1. Click "+ Add Target Reference Point" to insert a TRP between the spotter and the artillery.
  2. Each TRP requires a distance and azimuth measurement from the previous point in the chain (either the spotter or the preceding TRP).
  3. You can add multiple TRPs to create a chain: Spotter → TRP #1 → TRP #2 → ... → Artillery.
  4. When TRPs are present, the Artillery distance/azimuth fields change to measure from the last TRP to the artillery, rather than from the spotter directly.
  5. Altitude can optionally be entered for each TRP to improve accuracy over elevation changes.
  6. TRPs are removed in reverse order (last added, first removed) using the "- remove Target Reference Point" button.

Altitude Corrections

If the spotter, target, TRPs, or artillery are at different elevations, entering their altitudes allows the calculator to convert slant distances to horizontal distances for more accurate results.

Firing Plans

You can save firing plans for later use. To do this, add a new plan from the menu and enter a name. The current state will be persisted and can be reloaded by selecting it from the firing plan menu. The currently active plan is displayed and by pressing the button, you can reset the current values to the stored ones. Note that the correction portion is not restored.

Results

The In-game values are rounded to the nearest whole number. The Raw values show the unrounded computed result for reference.
